# SeeDAO 元规则第二版制定

## 事件概述

这是 SeeDAO 历史上最重要的治理事件之一。九人小组在 SeeDAO 经历多次低谷和治理危机后，总结过去 11 个月的经验教训，起草了这份元规则，为 SeeDAO 的自治奠定了基础。

## 背景

- SeeDAO Discord 服务器人数超过 8000 人
- 贡献者超过 600 人
- 内部已有十个公会，每个公会又有自己的项目
- 面临问题：谁是 SeeDAO 成员？公会、项目、成员和 SeeDAO 关系是什么？谁有权使用 SeeDAO 品牌？
- CryptoC 公司解散后，社区陷入「发起人独裁」的危险状态

## 制定过程

### 起草阶段
- 九人小组召开不下 **50 余次**会议
- 最长一次会议长达 **6.5 小时**
- 每人领一块任务起草
- 所有讨论公开透明

### 协商阶段
- 每个公会都派出协商小组
- 参与所有会议

### 社区讨论阶段
- 初稿开放给社区成员
- 每周两次晚上在 Discord 社区进行云会议
- 大家一起来讨论

### 修订阶段（v3）
- 2023 年 3 月
- somehacker 在逐字逐句翻译时发现表达、指称上存在不简练、不明确和不完善的地方
- 提出九处文辞调整建议
- 经乌鸦（宣传负责人）和唐晗（统稿人）讨论后形成修正案
- **重要原则**：仅限于文辞表述更精简明白，不涉及任何实质内容（权责利）的修改变动

## 治理架构

```
┌─────────────────────────────────────────────────────────┐
│              SeeDAO 节点共识大会（最高决策机构）              │
├─────────────────────────────────────────────────────────┤
│  市政厅（日常治理）        │    战略孵化器（经济发展）       │
├─────────────────────────────────────────────────────────┤
│              公会与项目（享有自决权）                        │
└─────────────────────────────────────────────────────────┘
```

## 核心创新

1. **就近治理 + 专家治理** — 让最接近问题的人做决定，让专业的人解决专业问题
2. **发起人远离决策** — 唐晗和白鱼作为发起人，不能最终拍板
3. **共识大会高门槛** — 要求过去三个月有活跃贡献，不能躺在历史功绩上
4. **分层规则体系** — I 级（元规则）、II 级（治理范围规则）、III 级（内部规则）
5. **POW 原则** — 贡献按照公认价值计算，而非简单的一人一票
